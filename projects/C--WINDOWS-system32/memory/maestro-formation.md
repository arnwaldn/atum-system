## Formules Notion (documentation officielle)

### Composants
- **Properties** : références aux champs de la base
- **Built-ins** : opérateurs (+, -, >, !=, and, or) et booléens
- **Functions** : actions retournant des résultats

### Fonctions clés
- `dateAdd(date, amount, unit)` — unités : years, quarters, months, weeks, days, hours, minutes
- `now()` — date/heure actuelle
- `replace(text, old, new)` — substitution texte
- `style(text, formats)` — formats : b, u, i, c, s + couleurs
- `length(list)` — nombre d'éléments ou longueur texte
- `first(list)` — premier élément
- `every(list, condition)` — vérifier condition sur tous
- `if(condition, then, else)` — conditionnel
- `sum(values)` — somme
- Ternaire : `X ? Y : Z`

### Exemples formation
- Due date : `dateAdd(Start Date, 2, "week")`
- Overdue : `if(and(now() > Due Date, Status != "Done"), "Overdue", "")`
- Comptage : `Tasks.length()`
- RICE : `Reach * Impact * Confidence / Effort`

## Méthodologies enseignées

### Double Diamant (Ch9)
Discover → Define → Develop → Deliver
- Phase divergente (explorer) → Phase convergente (focaliser)

### Priorisation (Ch10)
1. **100$ Bill** : distribuer 100$ fictifs entre features → révèle les priorités
2. **MoSCoW** : Must have / Should have / Could have / Won't have
3. **RICE** : (Reach × Impact × Confidence) / Effort = score
4. **Impact/Effort** : matrice 2×2, prioriser haut impact + faible effort

### Scoping projet (Ch11)
1. **User Story Mapping** : axe horizontal (parcours utilisateur) × axe vertical (détails)
2. Outils : Whimsical, Miro, FigJam
3. Gestion de projet : découpage en sprints/itérations

### Documentation (Ch14)
- Documenter dans chaque outil : Notion (hub projet), Make (notes modules), Airtable (descriptions champs)
- Template standard fourni par Maria Schools
